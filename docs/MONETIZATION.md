# 수익화 정책 — 어비스펫 (abysspet / 앱인토스)

검수·심의 제출용 요약. 구현: [`js/ads.js`](../js/ads.js), [`js/adConfig.js`](../js/adConfig.js).

## 수익 모델

| 유형 | 역할 | SDK |
|------|------|-----|
| **전면형(인터스티셜)** | 주 수익 | `loadFullScreenAd` / `showFullScreenAd` |
| **보상형** | 보조 수익 + 리텐션 | 동일 API, `userEarnedReward` 후 보상 |

1차 출시: **인앱 결제 없음**. 배너 광고 없음.

## 전면형 노출 (T1~T4)

| ID | 트리거 | 조건 |
|----|--------|------|
| T1 | 게임오버 → 새 펫 이름 입력 직전 | 쿨다운·세션 캡 통과 |
| T2 | 성체 진화 + 도감 등록 알림 이후 | 세션당 1회 |
| T3 | 졸업(새 알) 확인 후 이름 모달 직전 | 쿨다운·세션 캡 통과 |
| T4 | 오프라인 30분+ 복귀 메시지 이후 | 세션당 1회 |

**미노출:** 돌보기 직후, 진화 연출 중, 첫 세션 10분(튜토리얼 구간).

### 빈도 제한 (기본값, [`adConfig.js`](../js/adConfig.js)에서 조정)

- 세션당 최대 **3회**
- 최소 간격 **8분**
- 광고 실패 시 게임 플로우 계속 (fallback)

## 보상형 (R1~R3)

| ID | 보상 | UI | 제한 |
|----|------|-----|------|
| R1 | 사망 시점 펫 상태로 **부활** | 게임오버 «광고 보고 부활» | 사망당 1회 |
| R2 | 응급 돌봄 (배고픔+30 / 행복+25 / 청결+40 중 랜덤) | 스탯 최솟값 &lt; 40 | 세션당 3회 |
| R3 | 방치 타이머(`neglectStartedAt`) 초기화 | 평균 케어 &lt; 20 | 세션당 2회 |

보상은 반드시 **`userEarnedReward` 이벤트 이후**에만 지급합니다.

## 오디오

광고 재생 중 게임 SFX/Web Audio **일시 정지**, 종료 후 자동 재개 ([`js/audio.js`](../js/audio.js)).

## 데이터

- 펫·도감: `localStorage` (기기 로컬)
- 광고 세션 카운터: `sessionStorage`
- 사망 스냅샷: `sessionStorage` + 메모리 ([`js/deathSnapshot.js`](../js/deathSnapshot.js))
- 서버 전송 없음 (1차)

## 테스트

- 개발·샌드박스: `ait-ad-test-interstitial-id`, `ait-ad-test-rewarded-id`만 사용
- 운영 ID로 로컬 테스트 금지 (정책 위반)
- 브라우저: `?toss=1&mockAds=1` 로 UI 플로우만 검증

## 환불

광고만 사용 — 인앱 결제 환불 해당 없음.
